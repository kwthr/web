Esta guía está aquí para ayudarte a comparar rápidamente las características y sintaxis del framework CSS utility-first de Tailwind con las de Stylify, y darte una idea de cómo migrar de Tailwind a Stylify.

<MigrationNote />

## Introducción

Tailwind es una librería que te ayuda a construir rápidamente sitios web modernos sin salir de tu HTML. Es un framework CSS que prioriza la utilidad y está repleto de clases como flex, `pt-4`, `text-center` y `rotate-90` que se pueden componer para construir cualquier diseño, directamente en tu marcado.

## Selectores globales

<MigrationCodeBlock image="tailwind.svg">
<div slot="tool">

Cuando quieras seleccionar por ejemplo elementos hijo en Tailwind, puedes usar variantes arbitrarias para ello:

```html

	[&>*]:underline
	[&>p]:mt-0
```

## Componentes
<MigrationCodeBlock image="tailwind.svg">
<div slot="tool">

Los componentes en Tailwind se pueden definir usando la regla `@apply`. También se pueden anidar en capas CSS:

```css
@layer components {
	.btn-primary {
		@apply py-2 px-4 bg-blue-500 text-white;
	}
}
```

Tailwind también proporciona un montón de componentes copy&paste en su web.

## Configuración, personalización y variables
<MigrationCodeBlock image="tailwind.svg">
<div slot="tool">

Tailwind se puede configurar dentro de un archivo de configuración extendiendo/sobrescribiendo un tema existente:

```javascript
module.exports = {
	content: ['./src/**/*.{html,js}'],
	theme: {
		colors: {
			'blue': '#1fb6ff',
		},
		fontFamily: {
			sans: ['Graphik', 'sans-serif'],
		},
		extend: {
			spacing: {
				'8xl': '96rem',
				'9xl': '128rem',
			},
			borderRadius: {
				'4xl': '2rem',
			}
		}
	},
}
```
